# Настройка автоматических ежедневных коммитов

## Шаг 1: Создание GitHub репозитория

1. Создайте новый репозиторий на GitHub
2. Клонируйте его локально:
```bash
git clone https://github.com/MaxMad446/littleProject
cd littleProject

## Шаг 2: Настройка системы

1. Запустите setup.ps1 от имени администратора
2. Скрипт автоматически:
   - Проверит наличие Python и git
   - Создаст необходимые папки (daily_python_scripts, logs, backups)
   - Настроит задание в Планировщике задач
   - Создаст управляющие файлы

## Шаг 3: Управление системой

### Через менеджер (рекомендуется)
```powershell
# Запустите менеджер
./manager.ps1
```

### Через Планировщик заданий
1. Нажмите Win+R, введите `taskschd.msc`
2. Найдите задание `DailyGitHubCommit`
3. Управление:
   - Запустить: ПКМ -> Запустить
   - Остановить: ПКМ -> Остановить
   - Изменить время: ПКМ -> Свойства

## Шаг 4: Мониторинг

### Просмотр логов
```powershell
# Последние 20 записей
Get-Content "logs/daily_commit.log" -Tail 20
```

### Просмотр созданных скриптов
```powershell
# Последние 10 скриптов
Get-ChildItem "daily_python_scripts" -Filter "*.py" | Sort-Object LastWriteTime -Descending | Select-Object -First 10
```

## Шаг 5: Ручной запуск

```batch
# Запустите файл
run_daily.bat
```

## Шаг 6: Конфигурация

Файл `config.json` содержит настройки:
- `project_path`: путь к проекту
- `scripts_folder`: папка для скриптов
- `daily_time`: время ежедневного запуска
- `git_branch`: ветка Git
- `commit_message_prefix`: префикс сообщения коммита
- `ideas_file`: файл отслеживания идей
- `log_file`: имя файла лога

## Возможные проблемы

### Python не найден
- Убедитесь, что Python установлен
- Добавьте Python в PATH
- Перезапустите терминал

### Git не найден
- Установите Git for Windows
- Добавьте в PATH при установке

### Ошибки доступа
- Запустите PowerShell от имени администратора
- Убедитесь в наличии прав на запись

### Сетевые проблемы
- Проверьте интернет-соединение
- Убедитесь, что аккаунт GitHub авторизован

## Резервное копирование

Система автоматически создает резервные копии в папке `backups`:
- Каждый понедельник - полная копия репозитория
- Каждое 1-е число месяца - архив всех скриптов

```powershell
# Просмотр резервных копий
Get-ChildItem "backups" -Recurse
```

## Расширение функционала

### Добавление новых типов скриптов
1. Откройте `daily.py`
2. Найдите список `ideas`
3. Добавьте новую идею в формате:
```python
{
    "name": "new_idea_name",
    "code": "print('New script type!')\nprint('Custom functionality')\n",
    "category": "custom"
}
```

### Изменение расписания
1. Откройте Планировщик заданий
2. Найдите задание `DailyGitHubCommit`
3. Измените триггер на нужное время

## Поддержка

Если возникли проблемы:
1. Проверьте логи в `logs/daily_commit.log`
2. Запустите `manager.ps1` и выберите "Проверить настройки"
3. При необходимости обратитесь к разработчику
